# Codex Material Themes 1.0.2 修改交接

## 目標與目前狀態

版本 1.0.1 已公開發布，並已確認能在 ChatGPT 網頁安裝與執行完整的三階段流程：推薦主題、選擇字型、產生 `codex-theme-v1:` 設定及顯示匯入步驟。

目前唯一確認的功能缺陷是 Stage 1 預覽圖片在 ChatGPT 網頁中無法顯示。回覆只出現圖片替代文字，例如「午夜研讀預覽」。原因是 Skill 執行 `build-theme.mjs --preview-path` 後取得遠端執行環境的本機絕對路徑，再把該路徑嵌入 Markdown；使用者的瀏覽器無法存取該路徑。

公開網站已提供全部 8 張預覽圖。2026-07-21 已驗證以下代表網址回傳 `200 OK` 與 `Content-Type: image/png`：

`https://codex-themes.robinks.net/themes/Theme04.png`

## 1.0.2 設計決策

- 預覽圖片改用公開 HTTPS 網址，讓 ChatGPT 網頁與 Codex Desktop 都能顯示。
- 保留 Plugin 內附的 `assets/Theme01.png` 至 `Theme08.png`，作為本機資產及未來 fallback；不要從 ZIP 移除。
- Stage 1 不再執行 Node.js 取得本機 `--preview-path`。直接從主題目錄讀取明確的 `previewUrl` 並嵌入 Markdown。
- 保留 `build-theme.mjs`，繼續用於主題查找、資料驗證及確定性產生 `codex-theme-v1:`。不要改用 Python，也不要讓模型手工組裝設定 JSON。
- 不修改 8 組主題色彩、主題名稱、推薦規則、字型選項、三階段互動或匯入路徑。

## 實作內容

### 主題目錄與同步

1. 在來源目錄 `catalog/references/themes.json` 的每個主題加入 `previewUrl`：

   `https://codex-themes.robinks.net/themes/ThemeNN.png`

2. 更新 `scripts/sync-theme-catalog.mjs` 的驗證：

   - `preview` 仍必須符合 `../assets/ThemeNN.png`。
   - `previewUrl` 必須使用 HTTPS、指定正式網站網域，且檔名必須與 `preview` 相同。
   - 所有 `previewUrl` 必須唯一。

3. 執行同步，更新 Plugin 內的 `references/themes.json` 與網站的 `public/theme-catalog.js`。不要直接手改這兩個產物。

### Skill 行為

更新 `plugins/codex-material-themes/skills/choose-codex-theme/SKILL.md` 的 Stage 1：

- 讀取選定主題的 `previewUrl`。
- 使用 `![<theme name> preview](<previewUrl>)` 顯示預覽。
- 若介面無法內嵌遠端圖片，至少提供可點擊的 HTTPS 預覽連結，然後繼續原本的確認流程；不得只留下破圖替代文字。
- 移除 Stage 1 與 Commands 章節中的 `--preview-path` 使用說明。
- 保持「先顯示預覽，再詢問是否使用該主題」的 guardrail。

`build-theme.mjs` 可保留 `--preview-path` 供開發診斷，或移除該選項；它不得再成為公開 Skill 顯示預覽的必要條件。無論採哪一種，設定產生功能與輸出格式必須保持不變。

### 公開政策與版本

1. 將 `.codex-plugin/plugin.json` 版本由 `1.0.1` 更新為 `1.0.2`；package name、developer、URL、圖示及 prompts 不變。
2. 更新隱私權頁：

   - 將「只讀取內附預覽資產」改為「內附資產或公開網站託管的預覽圖片」。
   - 說明載入 HTTPS 預覽時，託管服務可能處理標準請求資訊，例如 IP 位址、User-Agent、時間與所請求的圖片路徑。
   - 說明不使用發布者營運的追蹤、分析或廣告，也不另行保存 prompts 或產生的設定。
   - 不再聲稱預覽請求完全不會揭露所選圖片；圖片檔名會出現在託管請求中。
   - 生效日期更新為實際公開 1.0.2 網站的日期。

3. 將條款頁的「previews bundled images」調整為「displays bundled or hosted preview images」，並把頁尾版本更新為 `1.0.2`。
4. 將支援頁的 Current release 更新為 `1.0.2`，並補充預覽無法顯示時可直接開啟 HTTPS 圖片連結。
5. 更新 `submission/submission-notes.md` 的版本與 1.0.2 修正摘要；既有正向／負向測試內容若仍符合行為，不需為改版而改寫。

## 驗證與驗收

### 自動與本機檢查

- `npm run check:themes` 通過，確認來源目錄與所有同步產物一致。
- `npm test` 通過，網站可成功建置。
- Skill validator 通過，YAML frontmatter、名稱及目錄結構有效。
- 對 Theme01–Theme08 的公開 URL 執行 HEAD 或 GET，全部回傳 `200`，且 `Content-Type` 為 `image/png`。
- 執行 `build-theme.mjs` 的列表及至少兩組設定產生測試，確認輸出仍是單行 `codex-theme-v1:`，色彩、字型與 JSON 結構未變。
- 檢查發布 ZIP 根目錄仍為 `.codex-plugin/` 與 `skills/`，並包含 8 張內附預覽圖；不得加入網站原始碼、`node_modules` 或 submission 文件。

### 跨介面手動測試

1. ChatGPT 網頁：

   - 安裝或更新至 1.0.2，建立新對話並從 `@` 選擇 Plugin。
   - 執行低眩光推薦 prompt，確認回覆直接顯示 Midnight Study 圖片，而非只有替代文字。
   - 說「換一個」，確認新主題對應的預覽同步改變。
   - 完成字型確認，確認設定字串及繁體中文匯入步驟仍正確。

2. Codex Desktop：

   - 安裝 1.0.2 後重新啟動 Codex，再執行相同流程。
   - 確認 HTTPS 預覽可顯示，三階段流程、字型選擇及設定輸出無回歸。

3. 失敗 fallback：

   - 模擬或暫時封鎖圖片網址時，回覆應提供可點擊的預覽 URL 並繼續詢問是否使用該主題，不得中止整個 Skill。

## 發布順序

1. 完成並驗證 Plugin 與網站變更。
2. 先重新發布公開網站，確認 8 張正式 URL 可讀及隱私／條款／支援頁已更新。
3. 建立乾淨的 `codex-material-themes-1.0.2-public.zip`。
4. 在 Plugin 發布頁上傳新版本並完成更新審核／發布流程。
5. 發布後在 ChatGPT 網頁與另一部 Codex Desktop 裝置重新安裝或更新，執行上述手動驗收。

## 不在 1.0.2 範圍

- 不新增、刪除或重新設計主題。
- 不修改 `codex-theme-v1` schema 或既有主題值。
- 不移除 Node.js 設定產生器，也不改用其他程式語言。
- 不嘗試自動修改 Codex 設定、AppData、資料庫或視窗。
- 不處理 Plugin Directory 的同步延遲；1.0.1 重新測試後已能正常載入與執行。
